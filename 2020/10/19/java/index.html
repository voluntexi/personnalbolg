JAVA为什么需要String args[]

发表于 2020-10-19 分类于 JAVA 阅读次数： Valine：

String args[]或者String[] args表示给主方法传一个字符串数组.

而args是一个字符串数组的变量名，不是关键字，

是arguments的缩写，只是一个默认名，一般都习惯性照写.

Q1:既然String args[]或者String[] args表示给主方法传一个字符串数组，那么

为什么是错误，args[]不就是那个字符串This is a simple Java program吗？

Args[]不和println有关

Args[]是保存运行main函数时候所输入的参数

例如main函数所在的类名为test 那么你在cmd运行 java test a b c 时，args[0] = a ,args[1]=b, args[2]=c 你就可以在你的程序中调用你输入的这些变量了。

Q2:如果没关系，为什么程序中去掉String args[]会出现错误？

java中不存在不带args[]的main函数，如果去掉这就不是主函数main了 所以出错

Java入口程序规定的方法必须是静态方法，方法名必须为main，括号内的参数必须是String数组。

因为java是一个面向对象的语言所以第一个要创建类

Public class XX

公共 定义类 类名

Public static void main（string[]args）

公共 静态 无返回值 入口函数

写了一个测试程序，在Eclipse上运行，以往都是点击一下运行就行了，今天试了几次都不行，然后把放到命令行去运行（cmd那个），编译：javac Test.java 运行：java Test hello world 显示：

是可以运行的，心想：Eclipse这么强大这个肯定可以办到，后来网上一查，其实很简单，就是打开Run里面的Debug configuration，里面的arguments里输入你想要输入的参数，然后保存，出来点击运行按钮，OK

结果出来了，显示的是刚刚输入的参数
